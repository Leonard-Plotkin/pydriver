The stereo Module¶

Module for extracting 3D information from stereo image pairs.

The class StereoReconstructor implements the reconstructing pipeline whereas OpenCVMatcher is an implementation of a specific disparity computation method.

Stereo¶

class pydriver.stereo.stereo.OpenCVMatcher¶

Disparity computation with OpenCV StereoSGBM class

Initialization raises an ImportError if cv2 can not be imported.

__init__(self, **kwargs)¶

Initialize OpenCVMatcher instance

Keyword arguments are the non-default parameters to pass to cv2.StereoSGBM().

The class uses its own default parameters which do not match OpenCV defaults.

Compute left disparity map for a pair of left and right images

Input images can have any type. They must be of same size and have 2 or 3 dimensions (3 channels if 3-dimensional).

class pydriver.stereo.stereo.StereoReconstructor¶

Pipeline for reconstructing 3D point clouds from stereo images

The class preprocesses the input images and then uses matchers (e.g. OpenCVMatcher) in the specified order to produce the final disparity map out of their results.

The function computeDisparity() is implemented so a StereoReconstructor can also be used as a matcher in another StereoReconstructor.

Required scene information: img_left, img_right, calibration[‘reprojection’].

Provided scene information: cloud, disparityMap_left, depthMap_left.

__init__(self, tuple matchers=None)¶

Initialize StereoReconstructor instance

Given matchers must implement the computeDisparity(img_left, img_right) function which returns the disparity map. Invalid disparity values must be negative (NOT NaN).

If matchers is not given, OpenCVMatcher() will be used.

Parameters:
matchers: tuple, optional

Instances of matchers to use, their results will be fused

Compute left disparity map for a pair of left and right images

This function allows a StereoReconstructor instance to be used as a matcher in another StereoReconstructor instance.
