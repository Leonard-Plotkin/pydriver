The detectors Module¶

class pydriver.detectors.detectors.Detector¶

Class for learning and recognition

This class implements learning features of different types and storing them in their respective vocabularies which can then be used to perform detection and recognition.

addWords(self, str category, str featureType, ndarray features, ndarray keypoints=None, dict box3D=None)¶

Add new visual words

The function adds features and their associated “meaning” (positive/negative example, object configuration if positive) to appropriate internal vocabularies which will be used for learning and recognition. Only features which are fully defined (no NaN values) are learned.

category: str
Category of object (or ‘negative’)
featureType: str
String indicating the feature type
features: np.ndarray[FLOAT_dtype, ndim=2]
Array with feature vectors
keypoints: np.ndarray[Position_dtype], optional
Array with keypoints, only for positive examples
box3D: dict, optional
3D box of object, only for positive examples, see BaseReader for box format description

class pydriver.detectors.vocabularies.Storage¶

Class for storing vocabulary data of specified category and recognizing it with a specified regressor

The class is aware of the special ‘negative’ category.

Produce detections for given features

class pydriver.detectors.vocabularies.Vocabulary¶

Vocabulary class to perform classifiction and recognition

The data for each category will be stored in its Storage instance.

Initialize vocabulary object

Parameters:
dimensions: int

Number of feature vector dimensions

preprocessors: list, optional

List of preprocessors which support fit() and transform() functions, empty by default

classifier: object, optional

Classifier object to use for category classification, sklearn.ensemble.AdaBoostClassifier(n_estimators=50) by default

storageGenerator: callable, optional

Function to use for creating new internal storage objects, must accept dimensions and category as first positional arguments, Storage(dimensions, category) by default

balanceNegatives: bool, optional

Flag whether to restrict the number of negative samples to maximum number of available samples for positive category, True by default
